# AGENTS.md

## 基本ルール

- コミットメッセージは日本語で書く。
- ユーザーの意図が実装挙動に関わる場合は、音楽・UI・状態遷移の解釈を勝手に広げすぎない。
- 変更前に既存の要件と実装を確認し、最小限の差分で直す。
- 作業後は可能な範囲で `npm run lint` と `npm run build` を実行して確認する。
- 未コミット変更がある場合は、ユーザーが明示的に頼んだ変更だけをコミットする。

## コマンドルール

- ユーザー入力が `大文字 + 半角コロン + 半角スペース` で始まる場合は、コマンドとして扱う。
- コマンド本文は、コロンとスペースの後ろに続く内容とする。
- 未定義のコマンドが使われた場合は、通常の依頼として扱うか、意図が曖昧なら確認する。
- コマンドは今後追加される想定のため、この一覧に追記して運用する。

### 定義済みコマンド

- `C: ...` は、本文に書かれた内容に従って Git コミットを行う。
- `E: ...` は、本文に書かれた対象や内容について説明を行う。

## プロジェクト方針

- このアプリは作業・集中用途のミニマルテクノ生成ツールであり、派手な曲生成サービスではない。
- 1小節内のパターンは固定して再生し、Mutation のタイミングでパターンや音色などを変化させる。
- 再生中に毎小節ランダムに音が抜けるような挙動は避ける。
- Mutation は一度に大きく変えず、1回につき局所的な変更に留める。
- `density` は再生時の確率ゲートではなく、Mutation 時のパターン密度調整として扱う。
- ブラウザの音声制約に従い、初回 Play 操作で Tone.js を開始する。

## 実装メモ

- 音声処理は client-side のみで行う。
- 状態管理は Zustand を使う。
- 保存対象は localStorage に置く。
- `node_modules/` と `.next/` は Git 管理しない。
- Windows 環境では UTF-8 の日本語ファイルが PowerShell 表示で文字化けすることがある。内容確認時は UTF-8 として読む。
