# Techno Daemon

自己進化するジェネレーティブ・レイヴ・マシン。ブラウザ上で音楽理論に基づいたミニマル〜レイヴ系テクノを無限に生成し、少しずつ手を加えて「育てる」ための Web アプリです。

一般的な「曲再生サービス」ではなく、微細な変化・永続ループ・ユーザーによる局所進化を重視しています。音声処理はすべてブラウザ内(client-side)で完結し、AI 推論やサーバー生成は使いません。

## 主な機能

- **ステップシーケンサー** — Kick / Snare / Hat / Bass / Synth の 5 トラック、16 ステップ・4/4・無限ループ。各ステップのオン/オフ、トラックごとのミュート・音量を編集可能。
- **音楽理論ベースの生成** — A マイナーを基調に、雰囲気ごとのコード進行(各 4 種、計 16 パターン)にベースとメロディーが追従。メロディーはスケール/ペンタトニック内に収まるよう制御。
- **Mutation(局所進化)** — 一定小節ごと、または手動で 1 箇所だけ変化。キックの 4 つ打ちやスネアのバックビートなどの土台は維持したまま、パターン・音色・フィルター・密度・強弱を少しずつ変える。
- **Accept / Revert とフィードバック学習** — 変化を採用/棄却でき、その選択から「どの種類の変化が好まれるか」を学習して以後の抽選確率に反映(localStorage に永続化)。
- **自動採用** — 指定したループ数だけ聴いたら未確定の変化を自動で確定。
- **Transform(曲調転換)** — 再生を止めずに 16 小節かけて、パターン・音色・メロディー・パラメータを新しいアレンジへ徐々にモーフ。コード進行も切り替わる。
- **雰囲気(Intent)** — coding / relax / dark / cyber を選択、またはテキスト入力のキーワードで反映。コード進行・メロディー音域・音色調整が変化。
- **画像トーン連動** — アップロード画像の平均色から明暗・色調を解析し、フィルターや密度へ反映。
- **日英切り替え / DAW 風ミニマル UI** — 言語切替(既定は日本語)、現在のキー・コード・進行・小節・学習状態をモニター表示。

## 技術スタック

- [Next.js](https://nextjs.org/)（App Router） / React / TypeScript
- [Tone.js](https://tonejs.github.io/) — Web Audio API 上のシーケンサー・シンセ・エフェクト
- [Zustand](https://github.com/pmndrs/zustand) — 状態管理（設定・学習重みは localStorage へ永続化）

## セットアップ

```bash
npm install
npm run dev      # 開発サーバー（http://localhost:3000）
```

その他のコマンド:

```bash
npm run build    # 本番ビルド
npm run start    # 本番サーバー
npm run lint     # ESLint
```

ブラウザの音声制約により、初回の Play 操作で音声エンジン（`Tone.start()`）が起動します。対応ブラウザは Chrome / Edge を優先。

## ディレクトリ構成

```text
src/
  app/          # Next.js のエントリ・レイアウト・グローバル CSS・アイコン
  components/   # メイン UI（シーケンサー / サイドパネル / モニター）
  audio/        # Tone.js の音声エンジン（シンセ・エフェクト・発音）
  hooks/        # 音声エンジンと状態を接続するフック
  store/        # Zustand ストア（状態・Mutation/Transform の適用・永続化）
  mutation/     # Mutation ルールと Transform（モーフ）ロジック
  theory/       # 音楽理論（キー・コード進行・スケール・移調）
  patterns/     # 初期パターンとデフォルト値
  i18n/         # UI ラベルの日英辞書
  types.ts      # 型定義
```

## 仕組みのメモ

- **Mutation は完全ランダムではなく安全な候補群から選ぶ**。各トラックのアンカー(常時オンのステップ)を維持し、変更は裏拍・シンコペーション位置に限定。密度は Euclidean リズムのヒット数へ写像。
- **フィードバック学習**は Accept で対象種類の選択重みを上げ、Revert で下げる(0.25〜4 倍でクランプ)。自動採用は「弱い好意」として控えめに加点。
- **Transform** は理論ベースで目標アレンジを生成し、残り小節数で差分を分割適用(連続値は線形補間、音色はトラックごとに別々の小節で切替)。
